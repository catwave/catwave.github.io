My Setup

Intro

Switching to a new laptop was a good opportunity to reconsider my whole setup. I chose Chimera as I like its design decisions.

“Chimera is an independent, general-purpose, rolling-release Linux distribution developed from scratch. It utilizes a FreeBSD-based userland, musl C library and the LLVM toolchain, along with the dinit service manager. Its primary focus is correctness, consistency and simplicity, but not at the expense of feature set”

I recommend reading their docs, it’s clear, concise, and more complete than this.

cbuild is also amazing.

Installing Chimera

Preparation

Download the iso, verify it and write it to a usb stick:

dd if=chimera-x86_64-live-gnome.iso of=/dev/sdX bs=1M status=progress oflag=sync

Partitioning (btrfs on LUKS)

Boot into the usb live environment, connect to internet and open a root shell. Default creds are anon:chimera.

Optionally erase all filesystem signatures

wipefs -a /dev/nvme0n1

Using cfdisk, create two partitions:

cfdisk /dev/nvme0n1
  1. efi, about 1 GB should be fine
  2. main, the rest of the space

Format the efi partition:

mkfs.vfat /dev/nvme0n1p1

In this setup, btrfs will replace both LVM and ext4.

Create LUKS device:

cryptsetup luksFormat /dev/nvme0n1p2
cryptsetup luksOpen /dev/nvme0n1p2 crypt

Format the main btrfs partition:

mkfs.btrfs /dev/mapper/crypt

Mount and create the btrfs subvolumes: (btrfs subcommands can be abreviated if there’s no ambiguity)

BTRFS_OPTS="rw,noatime,compress=zstd,discard=async"

mount -t btrfs -o $BTRFS_OPTS /dev/mapper/crypt /media

btrfs su cr /media/@
btrfs su cr /media/@home
btrfs su cr /media/@var
btrfs su cr /media/@swap
btrfs su cr /media/@snapshots

umount /media

Mount root, create the directories in which the subvolumes will be mounted, then mount the subvolumes:

mkdir /media/root

mount -t btrfs -o $BTRFS_OPTS,subvol=@ /dev/mapper/crypt /media/root

mkdir /media/root/boot
mkdir /media/root/home
mkdir /media/root/var
mkdir /media/root/swap
mkdir /media/root/.snapshots

mount -t btrfs -o $BTRFS_OPTS,subvol=@home /dev/mapper/crypt /media/root/home
mount -t btrfs -o $BTRFS_OPTS,subvol=@var /dev/mapper/crypt /media/root/var
mount -t btrfs -o $BTRFS_OPTS,subvol=@swap /dev/mapper/crypt /media/root/swap
mount -t btrfs -o $BTRFS_OPTS,subvol=@snapshots /dev/mapper/crypt /media/root/.snapshots
mount -o fmask=0077,dmask=0077 /dev/nvme0n1p1 /media/root/boot

Once everything is mounted, install Chimera and chroot into it:

chimera-bootstrap /media/root
chimera-chroot /media/root

Set the root password, install the stable kernel, add needed packages, create the swapfile, generate fstab, crypttab, and finally generate the initramfs:

passwd root

apk add linux-stable cryptsetup-scripts

btrfs filesystem mkswapfile --size 8g --uuid clear /swap/swapfile

swapon /swap/swapfile

genfstab / >> /etc/fstab

echo crypt /dev/nvme0n1p2 none luks,discard > /etc/crypttab

update-initramfs -c -k all

Install systemd-boot:

apk add systemd-boot
bootctl install
gen-systemd-boot

At this point, the system should be capable of booting on its own.

Post-installation steps

Create your user and add it to relevant groups:

useradd yourusername
password yourusername
usermod -aG wheel,kvm yourusername

Set hostname and time zone:

echo yourhostname > /etc/hostname
ln -sf /usr/share/zoneinfo/America/Montreal /etc/localtime

Enable logging system:

dinitctl enable syslog-ng

Desktop env

From this point, commands are executed as the normal user instead of root.

sway / swayfx

Copy dotfiles (note the trailing slash)

cp -r ./path/to/dotfiles/ ~

If not already in .ssh/config, create /etc/ssh/ssh_config.d/agent.conf with the contents:

Host *
  AddKeysToAgent yes

Add user repo:

doas apk add chimera-repo-user

Concatenate current world with this one, deduplicate, then install:

cat myworld /etc/apk/world | sort | uniq > newworld
vim newworld
doas mv newworld /etc/apk/world
doas apk fix

Extras

fprintd

doas apk add fprintd
doas fprintd-enroll yourusername

Insert the following line at the begining of system-auth pam file:

auth sufficient /usr/lib/security/pam_fprintd.so timeout=10

sed can be used to do this:

sed '1i\
auth sufficient /usr/lib/security/pam_fprintd.so timeout=10
' /usr/lib/pam.d/system-auth > newfile

After verifying the contents of newfile:

doas mv newfile /usr/lib/pam.d/system-auth

incus

The “incus-admin” group is instead named “_incus-admin”:

doas usermod -aG _incus-admin yourusername
# re-login
incus admin init

Unified Kernel Image (UKI) and UEFI Secure Boot

The goal here is to bundle the initrd, kernel, kernel commandline etc. into a single UEFI executable so that we only have to sign and verify a single item with secure boot.

doas apk add efibootmgr sbctl ukify

Reboot into the firmware interface, clear TPM keys and set the TPM into setup mode (for framework laptops, setup mode is called “TPM operation”)

loginctl reboot --firmware-setup

sbctl status should show “Setup Mode” as “Enabled”.

sbctl status

doas sbctl create-keys

doas sbctl enroll-keys

doas cp /proc/cmdline /etc/kernel/cmdline
  • edit /etc/kernel/cmdline as needed

Bundle and sign everything:

doas sh
kver=$(uname -r) ukify build --linux "/boot/vmlinuz-$kver" --initrd "/boot/initrd.img-$kver" --cmdline @/etc/kernel/cmdline --os-release @/usr/lib/os-release --output /etc/kernel/uki.efi

sbctl sign --output /boot/EFI/BOOT/uki.efi /etc/kernel/uki.efi

To manage boot entries from within the OS, efibootmgr can be used

To delete boot entry 0:

doas efibootmgr -B -b 0

To create the new boot entry (note the use of backslashes in the loader path)

doas efibootmgr --create --disk=/dev/nvme0n1 --part=1 --label="chimera" --loader='EFI\BOOT\uki.efi'

To auto-generate and sign the uki, this script can be used:

/etc/kernel.d/99-zz_uki.sh:

#!/bin/sh
# generate new uki as needed

uki=/etc/kernel/uki.efi
kver="$(linux-version list|linux-version sort --reverse|sed q)"

echo "Building uki for: $kver"

ukify build \
    --linux "/boot/vmlinuz-$kver" \
    --initrd "/boot/initrd.img-$kver" \
    --cmdline @/etc/kernel/cmdline \
    --uname "$kver" \
    --os-release @/usr/lib/os-release \
    --output "$uki"

sbctl sign --output /boot/EFI/BOOT/uki.efi "$uki" && rm "$uki" && echo "✓ Deleted $uki"

Verify uki signature:

doas sbctl verify

With the uki signed, secure boot can now be enforced.

TPM LUKS auto-decryption

doas apk add clevis-luks

Enroll using PCRs 0 and 7:

doas clevis luks bind -d /dev/nvme0n1p2 tpm2 '{"pcr_bank":"sha256","pcr_ids":"0,7"}'

Update initrd/uki:

doas update-initramfs -c -k all
doas /etc/kernel.d/99-zz_uki.sh

btrfs snapshots

Create root and home snapshots:

DATE=$(date "+%Y%m%dT%H%M") && \
btrfs sub snap -r / /.snapshots/root.$DATE && \
btrfs sub snap -r /home /.snapshots/home.$DATE

To restore, run from live env mount the toplevel filesystem to /mnt

mount -t btrfs -o rw,noatime,compress=zstd,discard=async /dev/mapper/crypt /mnt

then, rename the old root

mv /mnt/@ /mnt/@.old

And restore it from the readonly snapshot (note the absence of “-r”, so the new snap is rw)

btrfs sub snap /mnt/@snapshots/<name> /mnt/@
